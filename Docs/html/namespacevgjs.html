◆ schedule() [5/7]

template<typename T >
void vgjs::schedule ( std::pmr::vector< T > &  functions,
Job_base *  parent = current_job(),
int32_t  children = -1 
)
inlinenoexcept

Schedule functions into the system. T can be a Function, std::function or a task.

The parameter children here is used to pre-increase the number of children to avoid races between more schedules and previous children finishing and destroying e.g. a coro. When a tuple of vectors is scheduled, in the first call children is the total number of all children in all vectors combined. After this children is set to 0 (by the caller). When a vector is scheduled, children should be the default -1, and setting the number of children is handled by the function itself.

Parameters
[in]functionsA vector of functions to schedule
[in]parentThe parent of this Job.
[in]childrenNumber used to increase the number of children of the parent.
